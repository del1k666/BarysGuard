Варианты архитектуры многохостового сканирования

Выбери подход — или скажи в терминале что думаешь

A

Лёгкий агент + API-ключ

Агент (~200 строк Python/Go) запускается вручную или как сервис. HTTP + общий токен. Деплой — просто скопировать .exe. Быстро реализовать, легко дебажить.

Плюсы

  • Просто и быстро
  • Нет зависимостей
  • Легко тестировать

Минусы

  • Нет шифрования
  • Нет деплоя из UI
  • Токен хранится в открытом виде
B

Полный агент — Windows Service + HTTPS (рекомендую)

Агент устанавливается как Windows Service командой agent.exe --install. Генерирует self-signed сертификат. Главное приложение деплоит агент по WinRM. Полная интеграция со всеми вкладками через "хост-селектор" в шапке.

Плюсы

  • HTTPS — трафик зашифрован
  • Деплой из UI одной кнопкой
  • Работает после перезагрузки
  • Полная интеграция с вкладками

Минусы

  • Нужен WinRM на целевых хостах
  • Сложнее реализовать (~2 файла)
C

Агентless — WMI / PSRemoting

Никакой установки агента. Главное приложение само подключается к хостам через WinRM/WMI, копирует yara64.exe во временную папку, запускает скан и читает результаты обратно.

Плюсы

  • Ничего не нужно устанавливать на хосты
  • Встроенная Windows-аутентификация

Минусы

  • WinRM часто отключён в корпоративных сетях
  • Медленно (каждый раз копировать бинари)
  • Сложная обработка ошибок
  • Нет реалтайм-стриминга результатов
